## 🔌 Step 3: EC2 인스턴스 접속 (3분)

### Windows 사용자

#### 3-1. PuTTY 설치 (이미 설치되어 있으면 건너뛰기)

1. https://www.putty.org/ 접속
2. Download PuTTY 클릭
3. 설치 파일 다운로드 및 실행

#### 3-2. PuTTY 실행 및 접속

1. PuTTY 프로그램 실행
2. **Host Name** 입력창에:
   ```
   ec2-user@<복사한-IP-주소>
   ```
   예: `ec2-user@3.145.123.456`

3. 좌측 메뉴에서 **Connection > SSH > Auth > Credentials** 선택
4. **Private key file for authentication** 옆의 "Browse" 버튼 클릭
5. 다운로드한 `.ppk` 파일 선택
6. **Open** 버튼 클릭!

⚠️ 보안 경고가 나올 수 있습니다 → **"Yes"** 클릭

검은 화면이 나타나면 성공!

---

### Mac/Linux 사용자

#### 3-1. 터미널 열기
- Mac: Spotlight에서 "터미널" 검색
- Linux: Ctrl+Alt+T

#### 3-2. 키 파일 권한 설정 (최초 1회만)

다운로드 폴더로 이동:
```bash
cd ~/Downloads
```

키 파일 권한 변경:
```bash
chmod 400 kyuniv-pj-06-key.pem
```

#### 3-3. SSH 접속

```bash
ssh -i kyuniv-pj-06-key.pem ec2-user@<복사한-IP-주소>
```

예:
```bash
ssh -i kyuniv-pj-06-key.pem ec2-user@3.145.123.456
```

⚠️ "Are you sure you want to continue connecting?" → **yes** 입력 후 Enter

환영 메시지가 나타나면 성공!

## 🐛 문제 해결

### Q: 웹 페이지가 안 열려요!

**A: 3가지 확인:**

1. **EC2 인스턴스가 실행 중인가요?**
   - EC2 콘솔 확인
   - 상태가 "실행 중"이어야 함

2. **보안 그룹에 포트 5000이 열려있나요?**
   - EC2 콘솔 > 인스턴스 > 보안 > 보안 그룹
   - 인바운드 규칙에 포트 5000 확인

3. **서비스가 실행 중인가요?**
   - SSH로 접속
   - `sudo systemctl status ai-assistant` 실행
   - Active (running) 확인

---

### Q: AI 답변이 안 나와요!

**A: IAM 인스턴스 프로필 확인:**

1. EC2 콘솔 > 인스턴스 선택
2. 작업 > 보안 > IAM 역할 수정
3. `SafeInstanceProfileForUser-kyuniv-pj-06` 선택
4. 저장
5. 서비스 재시작:
   ```bash
   sudo systemctl restart ai-assistant
   ```

---

### Q: 배포 스크립트가 실패했어요!

**A: 인터넷 연결 확인:**

```bash
ping google.com -c 3
```

응답이 오면 인터넷은 정상입니다.

그래도 안되면:
```bash
cd ~/ai-learning-assistant
./deploy.sh
```
다시 실행해보세요!

---

### Q: SSH 접속이 안돼요!

**A: 확인사항:**

1. **키 파일을 사용하고 있나요?**
   - .pem 또는 .ppk 파일 경로 확인

2. **IP 주소가 맞나요?**
   - EC2 콘솔에서 퍼블릭 IP 재확인

3. **보안 그룹에 SSH가 열려있나요?**
   - 포트 22 인바운드 규칙 확인

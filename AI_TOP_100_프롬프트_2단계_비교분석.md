## 🔍 비교 분석 시 주의사항

### 1. 객관적으로 비교하기

**절대 하지 말아야 할 것**:
- ✗ Gemini 답안을 무조건 정답으로 간주
- ✗ 당신의 답안을 무조건 옳다고 주장
- ✗ 불일치를 무시하거나 얼버무리기

**반드시 해야 할 것**:
- ✓ 원본 데이터로 돌아가서 재검증
- ✓ 계산이 필요한 경우 다시 계산
- ✓ 양쪽 답안의 근거를 면밀히 비교
- ✓ 어느 쪽이 틀렸는지 명확히 판단

### 2. 재검증 방법

**문제 1 (메뉴 분석)**:
- 이미지를 다시 열어서 직접 카운트
- 칼로리 값을 엑셀이나 코드로 재계산
- "반찬명"의 정의를 명확히 하여 판단

**문제 2 (코드 석판)**:
- 코드를 다시 실행해보기
- 출력 결과를 바이트 단위로 비교

**문제 3 (영상 분석)**:
- 영상의 특정 시간대로 이동하여 재확인
- 자막이나 화면 텍스트 재확인

**문제 4 (전투 시뮬레이션)**:
- 데이터를 다시 집계하여 통계 재계산
- 승률 계산 코드를 다시 실행

**문제 5 (PDF)**:
- PDF를 다시 열어서 숨겨진 텍스트 재확인
- 여러 방법(색상 조정, 레이어 확인 등)으로 재탐색

### 3. 불일치 시 판단 기준

**우선순위**:
1. **원본 데이터**: 이미지, PDF, JSON 파일의 실제 내용
2. **재현 가능성**: 같은 방법으로 다시 해도 같은 결과가 나오는가
3. **논리적 타당성**: 답변의 근거가 논리적으로 타당한가
4. **상세도**: 더 구체적이고 상세한 분석이 이루어졌는가

**판단 시**:
- "Gemini가 맞을 것 같다"가 아니라 "데이터 재확인 결과 Gemini가 맞다"
- "내가 맞을 것 같다"가 아니라 "재계산 결과 내 답안이 맞다"

### 4. Gemini 오류 발견 시

Gemini도 AI이므로 오류가 있을 수 있습니다.

**오류 발견 시 작성 예시**:
```
❌ Gemini 오류 발견

문제 1-3 지역명 문제에서:
- Gemini 답안: "베트남, 전주"
- 실제 데이터 재확인 결과: "베트남(3회), 안동(2회), 전주(1회)"
- 전주는 1회만 등장하므로 오답
- 정답: "베트남, 안동"
- Gemini 오류 원인: 전주 카운트를 잘못 집계한 것으로 추정
```
